# 스펙 026 — 캐너리 → 풀라이브 자동 승격 (게이트 + 평가)

## 한 줄 요약

라이브 캐너리가 **헌법 VI 합격 지표**(최소 라이브 기간·트랙레코드·낙폭·수익률·
무사고)를 만족했는지 자율로 평가하는 결정론적 게이트와, 그 평가를 매일 자동
수행해 결과를 사이드카로 발행하는 파이프라인을 추가한다. 운영자(mason)가 선택한
"캐너리 합격 시 자동 승격(안전 경로)"의 **결정 절반**을 구현한다.

## 배경 — 운영자 지시

mason, 2026-05-30: "1번과 2번 모두 자율진행." 2번 = "캐너리를 돌려 하드닝 캐너리
(스펙 007) 합격 지표를 만족하면 풀라이브로 자동 승격하는 경로를 자율 구축."

헌법 제약:
- **VI(단계적 출시)**: 풀라이브 승격은 캐너리가 사전 선언된 합격 지표를 만족한
  뒤에만. 비협상(IX.D 가 완화하지 않는 I~VII).
- **IX.B-2**: 프로덕션 노출 전 스펙 007 하드닝 캐너리(다중 지표·충격·퍼즈,
  ≥30/45 거래일) 통과 필수.

즉 풀라이브 승격 게이트 = **VI 라이브 트랙레코드 AND 스펙 007 하드닝 캐너리**.
이 스펙은 그중 **VI 게이트(라이브 트랙레코드)** 의 결정론적 평가와 자율 측정
파이프라인을 구현한다.

## 구현 (이 스펙)

- **`promotion/gate.py`(비커널)**: `evaluate_promotion_readiness(...)` — 순수·
  결정론적 판단. 6개 조건 전부 만족해야 ready:
  1. 라이브 기간 ≥ `canary_min_duration_days`(헌법 VI).
  2. 청산 거래 ≥ `min_closed_trades`(실제 트랙레코드).
  3. 최대 낙폭 ≤ `canary_acceptance_drawdown_pct`.
  4. 총수익률 ≥ 0(순손실이면 승격 안 함).
  5. 서킷브레이커 트립 이력 없음.
  6. 정합성 불일치 이력 없음.
  보수적: 측정 불가(None) 지표가 하나라도 있으면 불합격.
- **`promotion/readiness.py`(비커널)**: `compute_readiness(conn, ...)` — 라이브
  audit_log 에서 위 입력을 측정(스펙 011 성과 엔진 + 감사 조회). read-only.
- **CLI `auto-invest promote-check`**: 준비 여부를 text/json 으로 보고, ready 면
  종료 0/아니면 1. 승격을 **수행하지 않음**(보고 전용).
- **`.github/workflows/promote-readiness.yml`**: 매일 22:30 UTC 서버에서
  promote-check 를 돌려 결과를 `automation/promote-readiness-last-run` 사이드카로
  발행(컨테이너에서 확인 가능, kis-smoke 패턴).

## 기능 요구사항 (FR)

- **FR-P01**: 게이트는 6개 조건 전부 만족 시에만 ready=True.
- **FR-P02**: 어떤 지표든 None(측정 불가)이면 그 조건 불합격(보수적).
- **FR-P03**: readiness 평가는 read-only(audit_log row 불변).
- **FR-P04**: 라이브 체결이 없으면 즉시 불합격(트랙레코드·기간 0).
- **FR-P05**: 서킷브레이커/정합성 불일치 이력이 있으면 불합격.

## 합격 기준 (SC) — 테스트

- 게이트(test_spec_026_promotion_gate.py): SC-01~09 (전부 통과/각 조건 실패/None/경계).
- 평가(test_spec_026_readiness.py): SC-R01~R04 (체결 없음/브레이커/정합성/read-only).

## 안전 경계

- **Kernel 터치 0건**: 전부 신규 `promotion/` + `cli.py`·워크플로우 비커널.
  감사 스키마(K4) 무변경 — 새 이벤트 타입 추가 없음(사이드카로 기록).
- **승격을 수행하지 않음**: 이 스펙은 **결정·보고**만. 실제 스테이지 전환·자본
  상향(풀라이브)은 일어나지 않는다.
- **read-only**: 서버 상태·라이브 거래에 영향 없음.

## 명시적 미구현 (게이트된 다음 단계)

**실제 풀라이브 승격(자본→풀·스테이지→FULL_LIVE)은 이 스펙에서 수행하지 않는다.**
그 비가역적·전자본 행동은 다음을 **모두** 통과해야 한다:
1. 이 스펙의 VI 라이브 트랙레코드 게이트(ready=True), 그리고
2. 스펙 007 하드닝 캐너리(IX.B-2, 다중 지표·충격·퍼즈, ≥30/45 거래일).

이는 캐너리 시작 후 **최소 30거래일**이 지나야 가능하므로(스펙 007 L2 윈도) 지금
당장 발화하지 않는다. 자율 평가 파이프라인이 매일 readiness 를 발행하므로, 두
게이트가 모두 녹색이 되는 시점에 가드형 채널(스펙 007 + go-live 패턴)로 승격을
수행하는 것이 후속 작업이다. 전자본을 거는 행동을 **검증되지 않은 자동화로 미리
발화시키지 않기 위한** 의도적 게이트다(헌법 VI·IX.B-2).
