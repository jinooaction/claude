# 스펙 025 — 다요인 합성 알파 점수 (Multi-Factor Composite Alpha Score)

## 한 줄 요약

유니버스 전체를 여러 팩터(모멘텀·퀄리티·저변동성·평균회귀)의 **횡단면 z-점수
가중합**(하나의 합성 점수)으로 순위 매겨, 상위 종목에만 매수를 허용하는 옵트인
필터를 추가한다. 스펙 021(모멘텀 단일 팩터)·023(퀄리티 단일 팩터)를 일반화한다.

## 배경 — 세계 최고 수준 격차

기존 시스템의 횡단면 선택은 **단일 팩터 필터를 순차 적용**한다:

- 스펙 021 `RankingFilter`: N-기간 수익률(모멘텀) 한 팩터로 상위 N개만 통과.
- 스펙 023 `QualityFilter`: 롤링 샤프/낙폭(퀄리티) 한 팩터로 상위 N개만 통과.

두 필터를 동시에 켜면 논리적으로 **AND**(둘 다 상위여야 통과)이다. 이 방식은
정보를 버린다 — 모멘텀 11위지만 퀄리티 1위·저변동성 1위인 종목은, 모멘텀 상위 10
필터에서 탈락한다(합성 점수로 보면 1위일 수 있는데도).

세계 최고 수준 계량 주식 전략(예: AQR 스타일 멀티팩터 합성)은 각 팩터를 유니버스
전체에 대해 **표준화(z-점수)** 한 뒤 **가중합**해 **하나의 합성 점수**로 순위를
매긴다. 이렇게 하면 "여러 면에서 두루 좋은" 종목이 "한 면에서만 극단적인" 종목보다
선호된다. 이것이 단일 팩터 순차 필터링 대비 가장 큰 알파 과학 격차다.

## 접근

새 모듈 `strategy/factors.py`(비커널)가 다음을 한다:

1. **팩터 원시값 추출**(심볼별, 가격 시계열만 사용 — 외부 재무 데이터 불필요):
   - `momentum`: N-기간 % 수익률(시계열 모멘텀, 높을수록 좋음). 스펙 018 재사용.
   - `quality`: 롤링 샤프 / (1 + |최대 낙폭|)(높을수록 좋음). 스펙 023 재사용.
   - `low_volatility`: **음의** 실현 변동성(저변동성 이상현상 — 변동성 낮을수록 높은
     점수). 스펙 017 `realized_volatility` 재사용.
   - `mean_reversion`: **음의** 볼린저 %B(과매도일수록 높은 점수). 스펙 018 재사용.
2. **횡단면 z-점수**: 각 팩터를 유한 원시값을 낸 심볼들에 대해 표준화
   (`(x - 평균) / 표준편차`, 모집단 표준편차). 표준편차 0이면 전부 0.
3. **가중합**: `합성 = Σ(가중치_f × z_f)`. 활성 팩터(가중치 ≠ 0)만 계산한다.
4. **결정론적 순위**: 내림차순, 동점은 심볼명 알파벳 순. 활성 팩터 중 하나라도
   유한 원시값이 없는 심볼은 센티넬 `Decimal("-Inf")`로 맨 뒤(데이터 부족 종목을
   데이터 완비 종목보다 절대 선택하지 않음 — 스펙 021/023과 동일 규약).

전부 결정론적 Decimal(6자리)이라 **라이브 = 백테스트 단일 잣대**(헌법 X.2).

## 기능 요구사항 (FR)

- **FR-C01**: `config/rules.py`에 `CompositeFactorFilter` 모델 추가
  (`universe`, `weights`, `lookback_bars`, `momentum_period`, `bb_period`,
  `bb_std`, `top_n` 또는 `top_pct`). `weights` 키는 허용 팩터 집합의 부분집합이어야
  하고, 최소 하나는 0이 아니어야 한다. `top_n`·`top_pct`는 정확히 하나만.
- **FR-C02**: `TradingRule.composite_filter: CompositeFactorFilter | None = None`
  추가. `None`이면 기존 경로 **byte 동일**(옵트인).
- **FR-C03**: `execution/order_router.py`가 퀄리티 필터 이후, 판단 자문 이전에
  합성 필터를 적용한다. 미통과 시 `SKIPPED_BY_COMPOSITE` 반환.
- **FR-C04**: `backtest/replay.py`가 같은 함수로 동일 필터를 적용하되, 각 세션
  날짜까지의 바만 사용해 **미래 참조를 방지**한다.
- **FR-C05**: `strategy/factors.py`의 `KNOWN_FACTORS`와 `config/rules.py`의
  `KNOWN_COMPOSITE_FACTORS`가 일치해야 한다(테스트로 동기화 검증).

## 합격 기준 (SC)

- **SC-01**: 같은 분산·다른 모멘텀에서, 모멘텀 가중치만 켜면 모멘텀 높은 종목이
  상위로 순위된다.
- **SC-02**: 두 팩터 가중치를 켜면, 한 팩터만 극단적인 종목보다 두 팩터 모두 양호한
  종목이 합성 점수에서 앞선다(합성의 핵심 동작).
- **SC-03**: z-점수는 평균 0·표준편차 1 성질을 만족한다(표준편차 0이면 전부 0).
- **SC-04**: 활성 팩터 중 데이터 부족 심볼은 센티넬로 맨 뒤에 위치한다.
- **SC-05**: `composite_filter=None`이면 기존 라우터·백테스트 경로와 byte 동일
  (회귀 무손상).
- **SC-06**: 옵트인 — `top_n`개 밖 심볼은 `SKIPPED_BY_COMPOSITE`로 차단된다.
- **SC-07**: 미래 참조 없음 — 백테스트가 세션 날짜 이후 바를 보지 않는다.
- **SC-08**: `KNOWN_FACTORS == KNOWN_COMPOSITE_FACTORS` 동기화.
- **SC-09**: `weights`가 알 수 없는 팩터를 포함하거나 전부 0이면 검증 오류.
- **SC-10**: 결정론 — 같은 바 입력이면 같은 순위(라이브 = 백테스트).

## 안전 경계

- **Kernel 터치 0건**: `risk/gates.py`(K1) 등 커널 무변경. 전부 `strategy/factors.py`
  신규 + `config/rules.py`·`execution/order_router.py`·`backtest/replay.py` 비커널.
- **하향 전용**: 필터는 주문을 **건너뛸 뿐**(스킵) 수량을 늘리지 않는다 — K1 캡이
  그대로 천장. 노출을 안전 경계 위로 절대 올릴 수 없다.
- **옵트인**: `composite_filter=None` 기본 → 기존 룰 byte 동일(마이그레이션 불필요).
- **결정론적 Decimal**: 백테스트 byte-equality + 라이브/백테스트 단일 잣대(헌법 X.2).
- **LLM 미사용**: 순수 수치 계산(헌법 III 무관).
- **dry-run 그대로**: 라이브 주문 경로 동작 변경은 필터(스킵)뿐. 실거래 전환과 무관.
- **외부 데이터 불필요**: 가격 시계열만 사용(재무 API 의존 없음).

## 검증

새 사이징/알파 작업은 반드시 `auto-invest walk-forward`(스펙 016 슬라이스 3)로
표본 외 검증할 것 — 합성 가중치가 한 기간에 과적합되지 않았는지 확인.
