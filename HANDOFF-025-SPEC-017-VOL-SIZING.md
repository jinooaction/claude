# HANDOFF 025 — 변동성 기반 포지션 사이징 (스펙 017 슬라이스 1, 2026-05-28)

PR #83 머지 커밋 `c291d75`. **"세계 최고 수준" 로드맵 — 측정 토대 다음 단계인 리스크
사이징을 시작**했습니다. 스펙 016 슬라이스 1~3이 백테스트를 정직(거래비용)·통일(단일
잣대)·표본 외 검증(워크포워드)되게 만들었다면, 스펙 017은 그 위에서 **포지션 사이징을
v1 고정 수량에서 변동성 인식으로** 올리기 시작합니다.

## 왜 (고친 갭)

측정은 정직해졌지만 사이징은 여전히 v1 수준이었습니다 — 룰마다 **고정 정수 수량**
(`Action.qty: int`)이 전부였습니다. 시장이 잔잔하든 폭락하든 같은 주식 수를 주문하고,
변동성·상관·리스크 패리티는 전무했습니다(유일한 동적 조정은 스펙 004 LLM 자문의
축소뿐).

세계 최고 수준의 시스템 트레이딩에서 **포지션 사이징(자금 관리)은 진입 신호 품질만큼
중요**합니다. 변동성 타깃팅(turbulent 구간에서 사이즈를 줄여 낙폭 완화)은 수십 년 증거로
뒷받침되는 **구조적 우위**이며, 신호 채굴과 달리 과적합 위험이 낮아 헌법 원칙 X(측정
기반·추측 금지)에 가장 잘 맞는 다음 단계입니다. 또 방금 통일한 잣대(샤프·최대낙폭)를
직접 끌어올립니다. 그래서 신호 과학보다 사이징을 1순위로 골랐습니다 — 리스크 사이징
토대를 먼저 깔아야 나중에 신호를 추가할 때 각 신호를 리스크 기여도로 사이징하고
워크포워드로 검증할 수 있는 규율 있는 순서가 됩니다.

## 한 줄 요약

- **새 모듈 `strategy/sizing.py`(비커널)** — 결정론적 Decimal:
  - `realized_volatility(closes)`: 연속 종가의 단순 수익률(r = cᵢ/cᵢ₋₁ − 1) 표본
    표준편차(분수 반환, 6자리 정규화). 수익률 < 2개(종가 < 3개)거나 비양수 종가면
    None(fail-safe 신호).
  - `volatility_scale(realized, target, min_scale=0)`: `min(1, target/realized)`을
    `[min_scale, 1]`로 클램프. 실현 변동성 ≤ 0이면 1(잴 위험 없음 = throttle 안 함).
  - `sized_quantity(base_qty, closes, sizing)`: `sizing`이 None 또는 mode="fixed"면
    기준 수량 그대로(v1 byte 동일). target_vol이면 최근 `lookback_bars` 수익률로 실현
    변동성을 구해 `floor(기준수량 × scale)` 반환. 실현 변동성 못 재면 기준 수량 반환
    (fail-safe). 결과 0이면 호출자가 "그 틱 건너뜀"으로 처리.
- **룰 스키마(`config/rules.py`, 비커널)** — 선택적 `SizingConfig`(`mode` fixed|target_vol,
  `target_volatility_pct` 기본 2.0, `lookback_bars` 기본 20, `min_scale` 기본 0).
  `TradingRule.sizing` 기본 `None`. None 또는 fixed면 v1 동작과 byte 동일이라 기존 룰
  TOML은 마이그레이션 없이 그대로 검증됩니다.
- **백테스트 `replay`와 라이브 `OrderRouter`(둘 다 비커널) 양쪽 연결** — 신호 발사 후
  K1 게이트 체인 **전에** `sized_quantity` 호출. 결과를 `OrderRequest.qty`·ORDER_INTENT
  감사·`OrderRecord`에 일관 사용. `sized < 1`이면 그 틱 건너뜀(`qty=0` 주문 미생성;
  라우터는 `SKIPPED_BY_SIZING` 반환). 라우터에서는 스펙 004 변동성 자문이 사이징 결과를
  기준으로 추가 축소(둘 다 축소만 하므로 K1은 그대로 바인딩).
- **안전 경계**:
  - **K1 캡 무변경 — 상한 그대로.** 사이저는 수량을 **제안만** 하고, 사이징 이후 K1
    게이트(`risk/gates.py`·`config/caps.py`)가 변형 없이 실행되어 그대로 바인딩.
    사이저는 노출을 K1 경계 위로 절대 올릴 수 없다. 슬라이스 1은 추가로 스케일 ≤ 1
    (하향 전용)이라 v1 기준 수량 대비 노출 증가 불가.
  - **Kernel 터치 0건.** 손댄 파일 전부 `strategy/sizing.py`(신규 비커널)·
    `config/rules.py`·`backtest/replay.py`·`execution/order_router.py`(비커널)·`tests/`·
    `specs/`. K1~K6·K-meta 무변경. **감사 스키마(K4) 무변경**(새 이벤트 0건,
    `SKIPPED_BY_SIZING`은 라우터 반환 상태 문자열일 뿐 감사 행 아님).
  - **결정론적·LLM 미사용.** 라이브·백테스트가 같은 결과(헌법 X.2). 라이브 워커 dry-run
    그대로(`AUTO_INVEST_MODE=live` 무관).
- **검증**: 테스트 신규 18건(전체 1086 통과, 4 스킵), 린트 깨끗. fixed/None 경로가 v1과
  byte 동일(기존 1068 테스트 무손상)으로 회귀 무손상(SC-S03/SC-S07) 증명. SC-S06은
  `run_walk_forward`를 target_vol 룰로 돌려 사이징이 워크포워드 표본 외 검증을 받음 확인.

## 다음 세션이 이어받을 것 (세계 최고 수준 로드맵)

리스크 사이징 1단계(하향 전용 throttle)가 깔렸습니다. 이제 양방향·멀티 포지션·신호로
확장하되, 워크포워드로 표본 외 검증을 받으며 안전하게:

- **슬라이스 2** — 양방향 타깃 변동성(잔잔한 구간에서 K1 봉투 **안에서** 사이즈 확대 →
  진짜 변동성 타깃팅) + 멀티 포지션 역변동성/리스크 패리티 배분. 주의: 확대를 허용하면
  K1이 유일한 하드 상한이 되므로, 사이저 출력이 항상 K1 게이트를 거치는지 재확인.
- **슬라이스 3** — 상관 인식 배분(상관 높은 종목 합산 리스크 한도).
- **신호/알파 과학** — 다요인(모멘텀·평균회귀·변동성)·레짐 인식·교차 단면 랭킹. 현재
  신호는 SMA·EMA·RSI + EMA 교차뿐(`strategy/indicators.py`·`triggers.py`).
- **후속 K4(선택)**: 사이징 결정(실현 변동성·스케일·최종 수량)을 감사 페이로드로 기록해
  포렌식 가시성 확보(K4 추가-전용 터치 1건).

**중요**: 새 사이징/알파 작업은 반드시 `auto-invest walk-forward`로 표본 외 검증을 거칠
것. WFE가 낮거나 표본 외 우위가 사라지면 그 변경은 과적합이므로 채택하지 말 것.
