## North star (operator-set, 2026-05-06)

The operator's stated long-term goal for this codebase:

> Build and operate a **world-class, self-improving automated
> investment service**. Investment scope is **not limited to US
> equities** — every domain that can be automated by a system is in
> scope (other equity markets, FX, crypto, derivatives where legal,
> rates, commodities, etc.).

Implications that bind every future spec:

1. **Measurement before money.** No new strategy reaches live capital
   until it passes a measured backtest with point-in-time correctness,
   realistic costs, and an out-of-sample window. This is constitution
   principle VI made concrete.
2. **Asset-class agnostic infrastructure.** Data, backtest, strategy,
   and portfolio modules are designed so adding a new asset class is
   a config + adapter change, not a rewrite.
3. **Self-improving loop.** The system should generate evidence about
   its own strategies (live vs backtest divergence, regime fit,
   parameter drift) and surface that evidence as input to the next
   research cycle. v1's daily report is the first primitive of this
   loop; later specs extend it.
4. **Honest scoping.** "Shipped" is not "done". Every milestone is
   evaluated against this north star, not against its own narrow
   scope.

When a spec or PR appears to optimise for short-term progress at the
cost of this north star (e.g. live-trading a strategy without a
backtest harness), the next session should call that out before
implementing.
